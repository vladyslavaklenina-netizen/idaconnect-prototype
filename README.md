# IDAConnect Screen Designs

Open `index.html` in a browser to review the clickable prototype.

## What Is Included

- A mobile screen-design prototype based on the existing IDAConnect prototype style.
- Family flow: Welcome, Create Account, Login, Add Loved One, Dashboard, Find Companion, Companion Profile, Book Visit, Payment Review, Confirmation, Visit Details, Visit Report, Plans.
- Companion flow: Verification, Companion Dashboard, Check-In, In-Visit, Submit Report.
- Future-role preview: Resident and Facility Staff coming-soon states.
- Clickable controls for role selection, screen navigation, filters, favorites, visit duration, visit purpose, consent, modals, and demo toasts.

## MVP Design Priorities

- Keep the loved one profile central before booking.
- Make verification and trust cues visible before payment.
- Treat visit reports as the core value screen.
- Keep resident and facility staff roles as future-facing unless the MVP scope changes.
- Avoid medical/clinical data; keep the product positioned around non-medical companionship.
